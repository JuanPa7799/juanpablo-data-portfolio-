Prediccion de churn para Interconnect

Solucion de punta a punta para identificar clientes con riesgo de cancelacion, integrando multiples fuentes, preparando datos, comparando modelos y comunicando resultados para toma de decisiones.

Python Pandas Scikit-learn Gradient Boosting AUC-ROC EDA

Resumen ejecutivo

El caso busca detectar clientes con alta probabilidad de abandonar el servicio para apoyar estrategias de retencion. La solucion integra cuatro fuentes de datos y selecciona un modelo supervisado con foco en AUC-ROC.

Modelo final Gradient Boosting
AUC-ROC test 0.8963
Accuracy test 0.8538

Datos

Se integraron 7043 filas usando `customerID` como llave. Las fuentes originales del ejercicio fueron: `contract.csv`, `personal.csv`, `internet.csv` y `phone.csv`.

Los datasets completos no se publican en esta version del portafolio para mantener una publicacion limpia.

Metodologia

  1. Carga y validacion inicial de calidad de cada fuente.
  2. Integracion de datasets por cliente.
  3. Definicion de la variable objetivo a partir de cancelacion.
  4. Limpieza, imputacion y codificacion de variables categoricas.
  5. Comparacion de baseline, Logistic Regression, Random Forest y Gradient Boosting.
  6. Seleccion final por AUC-ROC en test, con accuracy como metrica secundaria.

Resultados

  • Gradient Boosting fue el modelo seleccionado.
  • AUC-ROC en test: 0.8963.
  • Accuracy en test: 0.8538.
  • El flujo genera una base reproducible para segmentacion de riesgo y priorizacion de retencion.

Aprendizajes

El valor principal del proyecto esta en conectar un problema de negocio con una solucion tecnica completa: preparar datos de distintas fuentes, entrenar modelos comparables y explicar el resultado en terminos utiles para retencion de clientes.
