# Ruta Machine Learning Jr

## Posicionamiento

**Titulo objetivo:** Machine Learning Jr | Python | TensorFlow | Pipelines | Feature Engineering | Model Deployment

Esta ruta debe comunicar que puedes llevar modelos hacia una forma mas cercana a producto: pipelines, APIs, validacion de inputs, despliegue, logs y reproducibilidad.

## Dashboard del rol

El dashboard ML debe priorizar:

- modelos listos para servir;
- arquitectura;
- pipeline de entrenamiento;
- validacion de entradas;
- metricas de inferencia;
- proximos pasos de deployment.

### Proyectos actuales que pueden entrar

| Proyecto | Estado | Por que sirve |
|---|---|---|
| Calidad de energia | Proyecto estrella | ML aplicado a senales, precision 96%+ |
| Good Seed Age Vision | Fuerte | ResNet50, MAE 7.03 |
| IMDB Sentiment NLP | Fuerte | NLP, TF-IDF, LR, LightGBM, BERT sample |
| Rusty Bargain Pricing | Bueno | Comparacion calidad/latencia |
| Sweet Lift Taxi | Bueno | Feature pipeline temporal |

## Proyectos faltantes recomendados

1. API REST de prediccion con FastAPI.
2. Pipeline reproducible con sklearn Pipeline + joblib.
3. Mini dashboard de monitoreo de modelo.
4. Dockerfile y README de despliegue.

## Roadmap de habilidades

- Python para produccion: scripts, modulos, estructura de carpetas.
- FastAPI o Flask: `/predict`, `/health`, validacion con Pydantic.
- Pipelines: sklearn Pipeline, ColumnTransformer, serializacion.
- Deep Learning: TensorFlow/Keras, callbacks, exportacion de modelo.
- MLOps basico: versionado, logs, pruebas, Docker.
